API de Gestion des Récompenses

Plateforme d'attribution de récompenses pour les applications externes

Version v1.0.0

Introduction

Cette API permet aux plateformes externes d'attribuer des récompenses aux membres, de mettre à jour les soldes de ressources et de créer les transactions associées de manière automatisée.

Base URL

http://votre-domaine.com/api/rewards

Caractéristiques

  • Attribution automatique des récompenses
  • Création de transactions
  • Mise à jour des soldes
  • Support multi-entreprises
  • Validation complète

Sécurité

  • Authentification Laravel
  • Validation des utilisateurs
  • Vérification des entreprises
  • Journalisation complète
  • Gestion des erreurs

Attribuer une récompense

POST
/api/rewards/attribute

Permet d'attribuer une récompense à un membre. L'API se concentre uniquement sur l'attribution des points de récompense sans créer de transactions ou mettre à jour les soldes de ressources.

Paramètres

action_code Requis Code de l'action
supabase_user_id Requis* ID Supabase de l'utilisateur
user_id Requis* ID Laravel de l'utilisateur (alternative)
supabase_startup_id Optionnel ID Supabase entreprise
montant Optionnel Montant pour calcul
reference Optionnel Référence transaction
description Optionnel Description transaction

Note: Vous devez fournir soit supabase_user_id soit user_id, mais pas les deux. supabase_user_id est recommandé pour la compatibilité avec l'écosystème Supabase.

Réponse (200) 

{
  "success": true,
  "message": "Récompense attribuée avec succès",
  "data": {
    "recompense_id": 123,
    "recompense_valeur": 2500,
    "recompense_points": 2500,
    "membre_info": {
      "id": 1,
      "nom": "Doe",
      "prenom": "John"
    }
  }
}

Lister les actions disponibles

GET
/api/rewards/actions

Retourne la liste de toutes les actions de récompense disponibles avec leurs configurations. 

{
  "success": true,
  "message": "Liste des actions de récompense disponibles",
  "data": [
    {
      "code": "RECEPTION_SIKA",
      "titre": "Réception de Sika",
      "point": "25",
      "seuil": "25%"
    }
  ]
}

Vérifier les récompenses d'un membre

GET
/api/rewards/member/rewards?supabase_user_id={supabase_user_id}

Retourne toutes les récompenses d'un membre spécifique avec le total des points. 

{
  "success": true,
  "data": {
    "total_points": 7500,
    "nombre_recompenses": 3,
    "membre": {
      "id": 1,
      "nom": "Doe",
      "prenom": "John",
      "supabase_user_id": "supabase_12345"
    }
  }
}

Exemples d'utilisation

1. Attribution simple de récompense

Récompense sans montant (points fixes uniquement)

Requête:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "DEMANDE_SIKA",
  "supabase_user_id": "supabase_12345"
}

Résultat:

  • Récompense de 500 points créée
  • Aucune transaction créée (API simplifiée)
  • Aucune mise à jour de solde

1b. Attribution avec user_id (compatibilité)

Alternative utilisant directement l'ID Laravel

Requête:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "DEMANDE_SIKA",
  "user_id": "3c49fbe3"
}

Résultat:

  • Récompense de 500 points créée
  • Aucune transaction créée (API simplifiée)

Résultat:

  • Récompense de 500 points créée
  • Aucune transaction créée (API simplifiée)
  • Aucune mise à jour de solde

2. Attribution avec montant (calcul de pourcentage)

Récompense avec calcul de pourcentage basé sur le montant (pas de transaction)

Requête:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "RECEPTION_SIKA",
  "supabase_user_id": "supabase_12345",
  "supabase_startup_id": "startup_12345",
  "montant": 10000,
  "reference": "SIKA-2024-001",
  "description": "Financement SIKA reçu"
}

Résultat:

  • Récompense de 2500 points (25% de 10000)
  • Aucune transaction créée (API simplifiée)
  • Aucune mise à jour de solde
  • Association avec l'entreprise spécifiée

4. Nouveaux codes d'action disponibles

Exemples avec les nouveaux codes SOUMISSION_MARCHE, OBTENTION_MARCHE, QUIZ_70, QUIZ_90, MODULE_COURS

Requête - Soumission de marché:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "SOUMISSION_MARCHE",
  "supabase_user_id": "supabase_12345",
  "supabase_startup_id": "startup_12345"
}

Résultat:

  • Récompense de 1000 points créée
  • Aucune transaction créée (API simplifiée)
  • Association avec l'entreprise spécifiée

5. Codes Quiz

Exemples avec QUIZ_70 et QUIZ_90

Requête - Quiz 90% réussi:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "QUIZ_90",
  "supabase_user_id": "supabase_12345"
}

Résultat:

  • Récompense de 900 points créée
  • Aucune transaction créée (API simplifiée)

6. Module de cours

Exemple avec MODULE_COURS

Requête:

POST /api/rewards/attribute
Content-Type: application/json

{
  "action_code": "MODULE_COURS",
  "supabase_user_id": "supabase_12345",
  "description": "Module 'Introduction à l'entrepreneuriat' complété"
}

Résultat:

  • Récompense de 300 points créée
  • Aucune transaction créée (API simplifiée)

Actions de Récompense Disponibles

Code Titre Points Seuil Type Ressource Description
SOUMISSION_MARCHE Soumission de marché 1000 - Marché Points fixes pour soumission
OBTENTION_MARCHE Obtention de marché 5000 - Marché Points fixes pour obtention
QUIZ_70 Quiz 70% réussi 700 - Quiz Points fixes pour quiz réussi
QUIZ_90 Quiz 90% réussi 900 - Quiz Points fixes pour quiz réussi
MODULE_COURS Module de cours complété 300 - Formation Points fixes pour module complété
DEMANDE_CREDIT Soumettre une demande de Crédit 10000 - Crédit Points fixes pour demande
RECEPTION_CREDIT Réception de Crédit financier 10000 - Crédit Points fixes pour réception

Notes importantes

  • Les actions avec seuil en % calculent les points dynamiquement
  • Les actions sans seuil utilisent des points fixes
  • Un montant est requis pour les actions avec seuil de %
  • Les transactions sont créées uniquement si un montant est fourni
