API de Gestion des Paiements

Plateforme de déclenchement de paiements par actions pour les applications externes

Version v1.0.0

Introduction

Cette API permet aux plateformes externes de déclencher des paiements via actions prédéfinies, de créer automatiquement les transactions et de mettre à jour les soldes des ressources des membres de manière sécurisée et traçable.

Base URL

http://votre-domaine.com/api/payments

Caractéristiques

  • Déclenchement de paiements par actions
  • Création automatique de transactions
  • Mise à jour des soldes ressources
  • Support multi-entreprises
  • Validation complète
  • Gestion des priorités (2-3-4-1)

Sécurité

  • Authentification Laravel
  • Validation des utilisateurs
  • Vérification des entreprises
  • Journalisation complète
  • Gestion des erreurs
  • Transactions atomiques

Déclencher un paiement

POST
/api/payments/trigger

Permet de déclencher un paiement via une action prédéfinie. L'API crée automatiquement la transaction, met à jour le solde de la ressource et enregistre l'opération dans les logs.

Paramètres

action_code Requis Code de l'action de paiement
supabase_user_id Requis* ID Supabase de l'utilisateur
user_id Requis* ID Laravel de l'utilisateur (alternative)
supabase_startup_id Optionnel ID Supabase de l'entreprise
entreprise_id Optionnel ID Laravel de l'entreprise
diagnostic_id Optionnel ID du diagnostic (contexte)
montant Optionnel Montant personnalisé
reference Optionnel Référence personnalisée
description Optionnel Description personnalisée

Note: Vous devez fournir soit supabase_user_id soit user_id, mais pas les deux. supabase_user_id est recommandé pour la compatibilité avec l'écosystème Supabase.

Réponse (200) 

{
  "success": true,
  "message": "Paiement effectué avec succès",
  "data": {
    "payment_id": 456,
    "montant": 3000,
    "reference": "API-PREMIER-DIAG-123-20240409143000",
    "action_code": "PREMIER_DIAG_DIRIGEANT",
    "user_id": "user123",
    "supabase_user_id": "supabase_12345",
    "membre_id": 1,
    "entreprise_id": 456
  }
}

Lister les actions de paiement disponibles

GET
/api/payments/actions

Retourne la liste de toutes les actions de paiement disponibles avec leurs configurations directement depuis la base de données. 

{
  "success": true,
  "message": "Liste des actions de paiement disponibles",
  "data": [
    {
      "code": "PREMIER_DIAG_DIRIGEANT",
      "titre": "Réalisation du 1er diagnostic du dirigeant",
      "point": 3000,
      "seuil": "1",
      "description": "Premier diagnostic dirigeant"
    }
  ]
}

Vérifier le statut d'un paiement

GET
/api/payments/status/{reference}

Permet de vérifier le statut d'un paiement en utilisant sa référence unique. 

{
  "success": true,
  "data": {
    "reference": "API-PREMIER-DIAG-123-20240409143000",
    "status": "completed",
    "message": "Paiement traité avec succès"
  }
}

Exemples d'utilisation

1. Module de formation Pépite

Paiement pour complétion du module Pépite

Requête:

POST /api/payments/trigger
Content-Type: application/json

{
  "action_code": "MODULE_PEPITE",
  "supabase_user_id": "supabase_12345",
  "reference": "MODULE-PEPITE-001",
  "description": "Module Pépite complété"
}

Résultat:

  • Transaction de 1500 points créée
  • Solde du compte ressource Type 2 débité
  • Module ressource attribué
  • Log complet généré

1b. Module de formation Émergent

Paiement pour complétion du module Émergent

Requête:

POST /api/payments/trigger
Content-Type: application/json

{
  "action_code": "MODULE_EMERGEANT",
  "user_id": "3c49fbe3",
  "reference": "MODULE-EMERGEANT-002"
}

Résultat:

  • Transaction de 1200 points créée
  • Solde du compte ressource Type 2 débité
  • Module ressource attribué

2. Certification Elite

Paiement pour certification Elite avec entreprise

Requête:

POST /api/payments/trigger
Content-Type: application/json

{
  "action_code": "CERTIF_ELITE",
  "supabase_user_id": "supabase_12345",
  "supabase_startup_id": "startup_12345",
  "montant": 3000,
  "reference": "CERTIF-ELITE-001",
  "description": "Certification Elite obtenue"
}

Résultat:

  • Transaction de 3000 points créée
  • Entreprise trouvée via supabase_startup_id
  • Module ressource attribué
  • Montant personnalisé utilisé

3. Certification Pépite

Paiement pour certification Pépite

Requête:

POST /api/payments/trigger
Content-Type: application/json

{
  "action_code": "CERTIF_PEPITE",
  "supabase_user_id": "supabase_12345",
  "supabase_startup_id": "startup_12345"
}

Résultat:

  • Transaction de 2500 points créée
  • Type 2 utilisé (priorité 2-3-4-1)
  • Compatibilité ignorée (ressource_type_id = 0)

4. Gestion des erreurs

Exemple de réponse en cas d'erreur de validation

Requête invalide:

POST /api/payments/trigger
Content-Type: application/json

{
  "action_code": "",
  "supabase_user_id": "inexistant"
}

Réponse (422):

{
  "success": false,
  "message": "Données invalides",
  "errors": {
    "action_code": ["Le champ action_code est requis."],
    "supabase_user_id": ["L'utilisateur sélectionné n'est pas valide."]
  }
}

Actions de Paiement Disponibles

Code Titre Points Limite Type Description
MODULE_PEPITE Module Pépite 1500 Illimité Formation Module de formation Pépite
MODULE_EMERGEANT Module Émergent 1200 Illimité Formation Module de formation Émergent
MODULE_ELITE Module Elite 2000 Illimité Formation Module de formation Elite
CERTIF_PEPITE Certification Pépite 2500 Illimité Certification Certification Pépite
CERTIF_EMERGEANT Certification Émergent 2000 Illimité Certification Certification Émergent
CERTIF_ELITE Certification Elite 3000 Illimité Certification Certification Elite

Notes importantes

  • Les actions de modules sont illimitées et peuvent être utilisées plusieurs fois
  • Les actions de certifications sont illimitées et peuvent être utilisées plusieurs fois
  • Le système respecte la priorité 2-3-4-1 pour les comptes ressources
  • La compatibilité est ignorée pour ressource_type_id = 0
  • Toutes les transactions sont atomiques et journalisées
  • Les modules utilisent des badges de couleur teal, les certifications utilisent des badges indigo
